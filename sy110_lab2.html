Part 1

Navigating the File System. The Command Line Interface (CLI) may also be referred to as a terminal or console but it is a shell that accesses the Operating System (OS). Working with the shell can be intimidating but know that system adminstrators do not memorize all of the commands available and often refers to the help features within the CLI to conduct specific actions. This inital task will begin your familiarization with the shell and navigating through the file system.

Click on Windows Start and type powershell; Select Windows PowerShell to open the application; A new window will appear with the application name in the upper left corner displaying Windows PowerShell. Note the prompt in the CLI as this is your home directory.

The prompt will display your current directory and because you just accessed the CLI, the default directory will be your home directory. This next part will demonstrate several commands that will allow you to display the contents of a directory as well as traversing through the parent and child directories, up and down throughout the file hierarchy.

The root directory for all Windows OS's will be C:\. Absolute file paths always begin with C:\, which can be used anywhere in the entire file system.

From your home directory, display the contents for C:\Windows\Fonts by using the dir command. The proper syntax that would be entered in the CLI is dir C:\Windows\Fonts, where dir is the command and C:\Windows\Fonts is the argument.

This folder contains all of the fonts available on your system. Even though you are in your home directory, you were able to display the contents of the Fonts folder because you used the absolute file path. To change directories, simply use the cd command with the same path as you used in the previous step.

Once you enter the command in CLI, your prompt should reflect your new location. Enter the dir command without any arguments to display the contents of the folder.

When using .. as an argument for the dir command, it specifies your parent directory and . will indicate your current directory. Let's try this by typing dir . followed by dir .. to look at the contents of the current and parent folders, respectively.

Another shortcut to return to your home directory is ~\. Try using this by typing cd ~\ and observing the changes to your current directory. You should be back in your home directory of C:\Users\m289999. So far, you've navigated though the Windows file system using the CLI commands of dir and cd. An understanding of absolute and relative file paths should have also been applied. Next, we're going to look more into understanding commands within the shell.

Part 2

From the PowerShell CLI, you have the ability to execute programs just like you would in the Graphical User Interface (GUI). The help command may be useful in learning the command syntax, usage, and options available. In Windows, the help command can be accessed by the Get-Help command in PowerShell or -? option after a command. Let's look more into accessing help features and the CLI.

Open the old command prompt by typing cmd into your Windows search bar and hitting the Enter key. Note that you are still in your home directory but PS is no longer in the prompt. Type help in the CLI and observe the commands available in command prompt. Type exit to get out of command prompt and return to PowerShell. The prompt will begin with PS to indicate you are back in the PowerShell application.

Next, we'll use help to learn about another command. We'll take a look at the echo command. This command is useful for sending information to the screen.

Enter the command echo -? or Get-Help echo to observe details about the echo command. NAME displays the powershell name for this command. SYNTAX shows you how to use the command in powershell syntax. ALIAS shows commands that can be used in place of the powershell version of this command (e.g. they're interchangable).

Try using the command echo "Don't give up the ship!".

Commands like echo can be combined with additional commands via piping to perform useful functions such as writing text to a file or measuring properties of text. Enter the command echo "Don't Give Up the Ship!" | Measure-Object -Word and observe the output. Note that the output of echo was piped to the Measure-Object cmdlet with the -Word option used, resulting in the word count displayed. Learn more about the Measure-Object cmdlet by invoking the Get-Help command.

Part 5

The Windows Remote Desktop Protocol (RDP) is heavily used in the industry to access the GUI of another remote system. The access method covered in thes lab primarily focused on the Windows OSs shell, which is highly efficient for remote access connections. Bandwith needed for an SSH connection would be approximately 1,200 bits per second (bps) or the baud rate, which includes transmission and reception of any ASCII characters and overhead needed to secure the remote connection. Combining your knowledge of numeric systems in computing and remotely accessing another Windows GUI, let's look into the bandwidth requirements to appreciate the differences in managing systems architectures.

As you work through this problem, record your answers for the upcoming checkpoint question.

When looking into television or monitor screens, it's often advertised as 720p, 1080p, 4k, or even 8k. Let's take a look into what your screen is configured for.

Click on Windows Start and type settings.

Open the Settings app and under the System menu select Display. On the right, look at the Display Resolution to see what your monitor is configured for.

If your screen was set to 1920x1080, then your screen was set for 1080p - but wait...that's also considered 2k! 1920, or 2k resolution, sounds much better than 1080. The 'p' in 1080p stands for progressive instead of 1080i, or interlaced, which is the scan method used to update pixels across the screen but we're not going to get into that here. So we went from conveying the height of a screen resolution to the width, of which 4k is actually 3840x2160 pixels.

Now calculate how many pixels are present with a screen resolution of 1920x1080. (simply multiply the two numbers)

Each pixel represents a single dot or color that is measured in hex, with the color white being #FFFFFF and black being #000000. Think of all of the bits being turned on for white color (F) and all of the bits off for the absence of light, or black (0). Note that # just represents the values are of a hexidecimal color.

Each hex digit represents 4 bits (e.g. 0xF = 1111). So, 6 hex digits represents 24 bits or 3 bytes.

How much data (in bytes) are contained within a 1920x1080 image?

With a screen refresh rate of 30Hz, or 30 frames per second (FPS), how much bandwith (in bytes) is required to display a 2k image for one second? Each frame refresh draws every pixel on the screen.

Notice in the Computer Architecture lesson, a Hard Drive Disk (HDD) stores information in bytes. Network connections and data rates are measured in bits per second (bps). The bit-byte differences in storage and transmission rates are not recognizable by typical consumers. Just take a look at a any of the cellular data plans and see if they're measured in bits or bytes.

Part 1

SSH is a popular application that is used to connect to remote systems by implementing cryptographic protocols. It provides for confidentiality and authenticaion, unlike other insecure protocols like File Transfer Protofol (FTP), Trivial File Transfer Protocol (TFTP), and telnet. This task will guide you through the use of SSH on your Windows OS, the host system, in order to establish a secured connection to a Linux OS, the remote system.

Click on Windows Start and type powershell to open a console. The syntax of the ssh command only requires one argument, but it's a long one. The argument to ssh combines your username and the server that you are connecting to. Use the command ssh m289999@ssh.cyber.usna.edu where m289999 is replaced with your own alpha. When prompted, enter your usual password. **Characters will not appear as you are typing**

Confirm the name of the system you are interacting with by using the hostname command.

The SSH syntax will carry over to any OS you may be using and this course will be leveraging the ssh command to connect to various remote systems. This task introduced steps to remotely access another system using ssh and the information needed to login using your academy credentials.

Interaction across OSs are necessary to get information when using different systems. This is where commands may differ between OSs, specifically Windows and Linux. The following tasks will introduce commands to gather system information off a remotely connected system using the shell.

Verify the OS by using the command uname -o. View additional distribution info by using the command lsb_release -a and identify the codename.

The whoami command, similar to Windows, will display the username that is currently accessing the console. System commands in Windows and Linux OS may differ but PowerShell has programmed Linux commands as aliases in Windows!

Use lscpu to display the system architecture information (e.g., "Architecture").

The ps command displays the process status on the system. As a user, there might not be many processes you are currently running considering that you recently connected. Let's take a look by running ps and identifying the number of active processes.

Take a look at all active processes running on the server by adding some options to the command. Type ps -ef to see what's taking place.That was a lot of information that blew by. Controlling the output to screen is a technique system administrators have to master. Let's pipe the ps output to more by running the command ps -ef | more. Note that hitting the space bar will scroll down the information that can fit into the screen but using the enter key will scroll line-by-line. Typing q will quit more so you don't have to scroll through all 700+ lines.

Another way to look at system activity is with the top command. You should see plenty of other students logged in and using the system right now!Hint: Use q or Ctrl^C to exit the process.

A handful of commands will allow you to create, modify, move, and delete files off of your system. This task will demonstrate the use of common commands that will enable you to manipulate the file system using the CLI.

The prompt you are presented with is different than the PowerShell (PS) one displayed in Windows. Let's break down m289999@ward-rweb-09:~$m289999is your username. The pwd command prints the absolute path of your current working directory. Note that ~ is a shortcut to your home directory so you can use the tilde when invoking a directory path.

To list the contents of a directory, us the ls command. The cd command is the same in Linux as it was in Windows.

The touch command creates a file. Use this command to create a file named hello.txt. Verify the file now exists.

Use the echo command to add information to the hello.txt file: echo -n Hello > hello.txt. Review help to learn what the -n option does for the echo command.

In Linux, cat is used to output the contents of a file to the console. Use the cat command to display the contents of hello.txt to the console.

Create a folder named SY110 in your home directory using the mkdir command and referencing the .. relative file path. Recall from Windows that . is your current directory and .. is your parent directory and is the same in Linux.

Both hello.txt and world.txt should now be in the ~/SY110 directory. You may list the contents of your current working directory to verify.

The greater than symbol '>' sends the console output to a file, overwriting its contents. When using two greater than signs '>>' it will append that file. Use the cat command to view the contents of both the hello.txt file and world.txt file.

Previous actions copied the contents of hello.txt to world.txt and they should be the identical. Use the echo command to overwrite world.txt by entering the following into the console: echo World! > world.txt. View the contents of world.txt and notice that > overwrote Hello within the file with World!

Now append the hello.txt file with the contents of world.txt with the command: cat world.txt >> hello.txt Output the contents of hello.txt

Only delete the world.txt file using the rm command: rm world.txt

The commands leveraged as part of this task included pwd, ls, cd, touch, echo, cat, mkdir, cp, mv, rm, and rmdir. In addition, we expanded on using > for overwriting contents a file and >> for appending to a file. Using these commands in the CLI will enable you to manipulate the file system in the future when creating, modifying, moving, and removing files or directories within Linux.
